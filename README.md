# 芝しごと・楽RAC農薬ローテ

## 概要

「芝しごと・楽RAC農薬ローテ」は、芝管理の実務者が農薬ローテーションを判断しやすくするためのWebアプリです。  
農薬検索結果にRACコードを明確に表示し、同一グループを含む農薬や次回候補を比較できるように設計しています。

対象ユーザーは、ゴルフ場・公園・スポーツターフなどの維持管理に携わるグリーンキーパーや管理担当者です。  
現場で起きやすい「病害虫名では見つかるがローテーション判断が難しい」「同系統の重複使用を避けたい」といった課題の解決を目的としています。

## 主な機能

- RACコードの強調表示（FRAC / IRAC / HRAC）
- 農薬詳細画面でのRAC一覧表示（複数コード対応）
- 同一グループを含む農薬検索（一致RACの視覚強調）
- ローテーション候補表示（適用対象一致 + RAC重複回避）
- 芝・樹木・草花向けデータ抽出に対応
- PC/モバイル両対応のレスポンシブUI

## 開発背景

- FAMIC公開情報を土台に、現場利用を前提として再構築
- 「登録情報を見る」だけでなく「次の判断に使える」ことを重視
- グリーンキーパーの運用フロー（検索 → 比較 → 判断）に合わせて再設計

## 技術スタック

- フロントエンド: HTML / CSS / Vanilla JavaScript
- バックエンド: Node.js (`http.createServer` ベース、Express不使用)
- データ管理: JSONファイルをメモリロードして検索
  - `data/pesticides_target.json`
  - `data/pesticide_rac_target.json`
- デプロイ先: Render（予定）

## データについて

本サービスの農薬情報は、農林水産消費安全技術センター（FAMIC）公開情報を基に、実務向けに再編集した参考情報です。

## デプロイ方法（開発者向け）

### ローカル起動

1. 依存関係をインストール
   - `npm install`
2. サーバー起動
   - `node server.js`
3. ブラウザでアクセス
   - `http://localhost:3000`

### 環境変数（任意）

データファイルを切り替える場合のみ指定します。

- `PESTICIDES_JSON_PATH`  
  例: `c:\path\to\pesticides_target.json`
- `RAC_JSON_PATH`  
  例: `c:\path\to\pesticide_rac_target.json`
- `PORT`（任意）  
  例: `3000`

### Renderデプロイ概要

- リポジトリをGitHubへ公開
- RenderでWeb Serviceを作成し、対象リポジトリを接続
- Build Command: `npm install`
- Start Command: `node server.js`
- 必要に応じて環境変数（上記）を設定
- デプロイ後、ログでJSONロード件数を確認
